# Project Docs

Use this folder as the entry point for working on the site. The docs are organized by topic so you can ramp up quickly:

- [Architecture](architecture.md) — stack, directory map, data flow, conventions.
- [Content & Routing](content-and-routing.md) — Markdown model, route map, language/category rules, build inputs.
- [Components](components.md) — key UI building blocks and how pages compose them.
- [Development Guide](development.md) — setup commands, styling notes, utilities reference, and common tasks.

If you add new features, keep this folder updated so future contributors can rely on docs first before diving into code.
